# Voca Study (Web)

기존 윈도우 tkinter 앱(Voca Study)을 웹/모바일 브라우저용으로 옮긴 버전입니다.
데이터는 더 이상 로컬 PC가 아니라 **GitHub 저장소**에 JSON 파일로 저장되며,
폰/PC 어디서든 같은 저장소에 접속해서 이어서 학습할 수 있습니다.

## 1. GitHub 저장소 준비

1. GitHub에서 새 저장소를 만듭니다. 이름은 자유지만 `vaca_study` 추천 (Public/Private 둘 다 가능).
2. 저장소 루트에 다음 파일들을 올립니다.
   - 이 폴더의 `index.html`, `style.css`, `app.js`
   - 기존에 쓰던 데이터 파일: `dictionary.json`, `scores.json`, `deleted_words.json`, `weekly_stats.json`
   - (`config.json`은 윈도우 창 위치 저장용이라 웹에서는 필요 없습니다 — 안 올려도 됩니다.)

## 2. GitHub Pages로 호스팅

1. 저장소 **Settings → Pages**
2. **Source**: `Deploy from a branch` 선택
3. **Branch**: `main` / 폴더는 `/ (root)` 선택 후 Save
4. 1~2분 후 `https://<사용자명>.github.io/<저장소명>/` 주소가 생성됩니다. 이 주소를 폰 브라우저에서 열면 됩니다 (홈 화면에 추가하면 앱처럼 사용 가능).

## 3. Personal Access Token 발급 (최초 1회, 기기별로 1번씩)

진짜 OAuth는 클라이언트 시크릿을 안전하게 둘 서버가 필요해서, 정적 페이지에서는
**Fine-grained Personal Access Token**을 한 번 발급해서 앱에 입력하는 방식으로 "권한 부여"를 구현했습니다.
토큰은 브라우저(`localStorage`)에만 저장되고 외부로 전송되지 않습니다 — GitHub API 호출에만 사용됩니다.

1. GitHub → 우측 상단 프로필 → **Settings → Developer settings → Personal access tokens → Fine-grained tokens**
2. **Generate new token**
3. **Repository access**: `Only select repositories` → 방금 만든 저장소만 선택
4. **Permissions → Repository permissions → Contents**: `Read and write` 로 설정
5. 발급된 토큰(`github_pat_...`)을 복사

> ⚠️ 토큰은 비밀번호와 같습니다. 캡쳐/공유하지 마세요. 새 기기에서 쓰려면 같은 토큰을 한 번 더 입력하면 됩니다(또는 새 토큰 발급).

## 4. 앱 첫 접속

1. Pages 주소 접속 → "GitHub 연동 설정" 화면이 나옵니다.
2. 사용자명, 저장소 이름(`vaca_study`), 브랜치(`main`), 토큰 입력 → "저장하고 시작하기"
3. 저장소 연결이 확인되면 바로 단어 학습이 시작됩니다.

## 동작 방식 / 원본과의 매핑

| 원본 (tkinter)              | 웹 버전                                  |
|------------------------------|-------------------------------------------|
| `dictionary.json` 로컬 파일  | GitHub Contents API로 읽기/쓰기            |
| `scores.json`                | 동일 (★/X 변경 시 1.5초 디바운스 후 자동 커밋) |
| `deleted_words.json`         | 단어 삭제 시 기록, dictionary/scores에서도 제거 |
| `weekly_stats.json`          | 주차 계산 알고리즘 동일 포팅 (일요일 시작 기준) |
| 가중치 랜덤 출제              | `1.0 + ★×2.0 − X×0.5` (최소 0.1) 동일       |
| ★+/-, X+/-, 삭제, 복사 버튼   | 동일 동작, 모바일 터치 버튼으로 재구성       |
| 키보드: →/Space, +/-, X/C, R, V | 동일 (PC 브라우저에서 작동, 폰은 버튼 사용) |
| 창 위치 저장 (`config.json`) | 해당 없음 (브라우저 창 개념 없음)           |

## dictionary.json 이 계속 보강되는 경우 (외부 편집)

`dictionary.json`을 앱 밖에서 직접 편집/보강해서 push하는 경우를 고려해서 다음과 같이 동작합니다.

- **읽기**: 앱이 데이터를 불러오는 모든 시점(첫 로드, ↻ 새로고침, 탭 재활성화 시 자동 재동기화)에
  `dictionary.json`을 **항상 GitHub에서 새로 받아옵니다.** 새 단어 추가나 풀이 보강은 다음 로드 때 그대로 반영됩니다.
- **단어 삭제**: 앱에서 🗑️로 단어를 지울 때, 로컬에 들고 있던 캐시를 기준으로 통째로 덮어쓰지 않습니다.
  대신 삭제를 실행하는 시점에 **GitHub의 최신 `dictionary.json`을 다시 받아온 뒤, 그 위에서 삭제할 단어만 제거**해서
  저장합니다. 그래야 그 사이에 외부에서 보강해둔 다른 단어들의 변경 내용이 삭제 작업 때문에 사라지지 않습니다.
- 단, **정확히 같은 단어**를 외부 편집과 앱 삭제가 초 단위로 동시에 건드리는 극단적인 경우까지는 보장하지 않습니다
  (그 정도의 동시성은 현실적으로 거의 발생하지 않습니다).



- 데이터는 **브라우저 캐시에 저장되지 않습니다.** `localStorage`에는 GitHub 토큰/저장소 설정만 저장되고,
  단어/점수/주차 데이터는 앱을 열 때마다 GitHub에서 새로 받아오고, 변경 시마다 GitHub에 다시 올립니다.
  그래서 폰에서 수정한 내용을 데탑에서 (새로 열거나 ↻ 버튼으로) 그대로 받아볼 수 있습니다.
- **저장은 즉시가 아니라 1.5초 디바운스 후** GitHub에 커밋됩니다. 버튼을 여러 번 빠르게 누르면
  한 번의 커밋으로 합쳐서 올라가요(API 호출 절약). 화면 상단 동기화 표시(●)로 상태 확인 가능.
- 탭을 백그라운드로 보내거나(다른 앱 전환) 닫을 때 보류중인 저장을 즉시 시도합니다
  (`visibilitychange` / `pagehide` / `beforeunload` 모두 등록). 다만 모바일 브라우저 특성상
  100% 보장은 아니라서, **연속으로 여러 번 조작한 직후엔 잠시 화면에 머물러 "저장됨" 표시를 확인하고
  넘어가는 걸 권장**합니다.
- 탭을 다시 활성화하면(다른 앱 갔다 돌아오기) 보류중인 변경사항이 없을 때 **자동으로 GitHub에서
  최신 데이터를 조용히 재동기화**합니다. 우측 상단 ↻ 버튼으로 언제든 수동 새로고침도 가능합니다.
- 같은 데이터를 **여러 기기에서 정확히 동시에** 수정하면(초 단위로) 마지막에 저장한 쪽이 이길 수 있습니다.
  순서대로 쓰면(폰에서 끝내고 데탑 켜기) 문제 없습니다.
- 토큰 권한 범위를 해당 저장소 1개 + Contents 권한만으로 최소화했지만, 토큰을 아는 사람은 그 저장소에 쓰기가 가능하니
  타인에게 페이지 링크와 토큰을 함께 넘기지 마세요. 토큰을 입력하지 않으면 데이터는 보이지 않습니다(읽기 전용 공유 X).
- 토큰을 바꾸거나 다른 저장소로 옮기려면 화면 우측 상단 ⚙️ 버튼으로 설정 화면을 다시 열 수 있습니다.

## 로컬에서 테스트하기 (선택)

```bash
cd vaca_study_web
python3 -m http.server 8000
# http://localhost:8000 접속
```
GitHub Pages에 올리기 전에 로컬 서버로 먼저 동작을 확인할 수 있습니다(단, GitHub 저장소/토큰은 실제로 필요).
